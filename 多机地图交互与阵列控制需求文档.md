# 多机地图交互与阵列控制需求文档

## 1. 文档目的

本文档用于在现有 `UE5DroneControl` 项目基础上，明确下一阶段的多机交互、地图加载与阵列控制需求，作为后续设计、开发、测试和验收的统一依据。

本次需求重点包括：

1. 优先接入 `Cesium for Unreal`，实现流式加载地图
2. 支持多机状态下的无人机选中与地图点选派发
3. 支持中键查看无人机信息面板
4. 支持多机状态下的自由镜头模式
5. 支持无人机阵列移动序列编辑与播放

---

## 2. 建设目标

本阶段的目标不是一次性做成完整的无人机任务系统，而是先构建一个可用、可演示、可扩展的基础能力层，满足以下诉求：

- 让 UE5 场景从固定平面环境升级为可流式加载的真实地图环境
- 让操作员可以在多机情况下快速选择单架无人机并进行点位派发
- 让操作员可以在运行时查看单架无人机的关键信息，并为未来接入更多遥测字段预留扩展空间
- 让操作员在多机观察时可以脱离跟随机制，基于当前视角方向自由浏览整个空域和任务区域
- 让系统具备基础阵列运动编排与播放能力，用于演示和验证多机协同效果

---

## 3. 需求范围与优先级

| 优先级 | 编号 | 需求名称 | 说明 |
|------|------|------|------|
| P0 | FR-01 | Cesium for Unreal 接入与流式地图加载 | 为其余功能提供真实地图底座，优先级最高 |
| P1 | FR-02 | 多机点选无人机并点击地图派发目标点 | 多机核心交互能力 |
| P1 | FR-03 | 中键悬停弹出无人机信息 UI | 运行态信息查看能力 |
| P1 | FR-04 | 多机自由镜头模式 | 多机态势浏览能力 |
| P2 | FR-05 | 阵列移动序列编辑与播放 | 多机演示与协同运动能力 |

## 4. 典型使用场景

### 4.1 地图加载场景

操作员启动项目后，系统自动接入 `Cesium for Unreal`，根据预设服务地址加载三维地图数据。地图采用流式方式按视口与距离逐步加载，避免一次性载入全部场景。

### 4.2 单机派发场景

操作员在地图上点击某架无人机以选中目标，无人机进入高亮状态；随后再次点击地图上的某一点，系统将该点作为目标位置派发给当前选中的无人机。

### 4.3 信息查看场景

操作员将鼠标悬停在某架无人机上，按下鼠标滚轮中键，弹出该无人机的状态信息面板，用于查看当前基础信息以及未来扩展后的遥测数据。

### 4.4 多机观察场景

在多机运行时，操作员可切换为自由镜头，不再跟随某一架无人机，而是在地图范围内基于当前视角方向进行前后左右相对移动并观察多机态势。

### 4.5 阵列演示场景

操作员对若干无人机编辑一段阵列动作序列，设置队形、偏移、时间和播放顺序，随后执行播放，系统按预设驱动无人机完成简单阵列运动演示。

## 5. 功能需求

### FR-01 实现项目对 Cesium for Unreal 的支持，实现流式加载地图

### 5.1 目标

为项目接入真实三维地图能力，使无人机控制、信息展示和阵列演示不再依赖固定测试地形，而是在可流式加载的地图环境中运行。

### 5.2 功能说明

系统需支持在 Unreal 项目中集成 `Cesium for Unreal`，并通过配置的地图服务地址、工程地址或数据源加载三维地图内容。地图加载应采用流式方式，根据镜头位置、视野范围和层级进行动态加载与卸载。

### 5.3 详细需求

#### 5.3.1 SDK 接入

- 项目需支持引入并启用 `Cesium for Unreal`
- 项目启动时应能正确初始化 SDK 相关组件
- 地图加载所需配置应支持通过配置文件、蓝图参数或编辑器可配置项进行设置
- 应支持至少一个默认地图场景的自动加载

#### 5.3.2 地图流式加载

- 地图数据应按需加载，不要求一次性加载全部场景
- 当镜头移动、缩放或切换观察区域时，地图内容应随之动态补充
- 对当前视口不可见或距离较远的地图数据，应允许进行降级显示或卸载处理
- 地图加载过程应尽量避免长时间卡顿或明显阻塞主线程

#### 5.3.3 坐标与场景对齐

- 需明确 `Cesium for Unreal` 地图坐标系与项目当前无人机坐标系之间的转换关系
- 无人机在地图中的位置应与地图底图对齐，不出现明显偏移、倒置或方向错误
- 地图原点、比例尺、朝向和高度基准应形成统一规则
- 后续多机目标点派发和阵列编辑均应复用该坐标转换规则

#### 5.3.4 可视化与交互

- 地图加载时应提供基本状态反馈，例如“加载中”“加载完成”“加载失败”
- 无人机模型应能正常叠加显示在地图之上
- 当地图尚未加载完成时，用户仍应看到基础占位反馈，而不是空白黑屏
- 若地图服务不可用，系统应给出明确错误提示

#### 5.3.5 兼容性与扩展性

- 地图加载能力应兼容后续多机控制、信息 UI、自由镜头和阵列编辑功能
- 后续如需切换地图服务地址或加载其他区域，不应要求改动核心业务逻辑
- 需预留后续叠加航点、轨迹线、禁飞区、区域标注等地图元素的能力

### 5.4 验收标准

- 项目启动后可成功加载一套基于 `Cesium for Unreal` 的三维地图
- 地图在移动视角过程中能持续流式加载，不出现整图长期空白
- 无人机能在地图上正确显示，位置与方向基本正确
- 地图服务异常时，界面可给出可识别的错误反馈
- 后续其他功能在地图加载后可继续正常工作

### FR-02 多机通讯时，点击无人机选中后，再点击地图可指定该无人机飞向目标点

### 5.5 目标

建立多机控制下的基础交互闭环，让操作员可以通过“先选无人机，再点地图”的方式，向指定无人机下发目标点。

### 5.6 交互流程

1. 操作员用鼠标点击某一架无人机
2. 系统识别点击命中对象为无人机，并将其设置为当前选中目标
3. 被选中的无人机需有明显视觉反馈，例如描边、高亮、图标变化或状态标签
4. 操作员再次点击地图上的可达位置
5. 系统将该地图坐标转换为目标点，派发给当前选中的无人机
6. 系统反馈目标点已设置，并可选显示目标标记、连线或提示文本

### 5.7 详细需求

#### 5.7.1 选中逻辑

- 任意时刻最多允许一架无人机处于“主控选中”状态
- 当点击新的无人机时，原选中目标应被取消，新目标成为当前选中对象
- 若点击空白区域且未触发地图派发逻辑，可保持当前选中状态不变
- 若当前没有选中无人机，点击地图不应直接下发目标点

#### 5.7.2 地图派发逻辑

- 只有在已有选中无人机的前提下，点击地图才会触发目标点派发
- 地图点击点需通过射线检测或地理坐标转换获得三维目标坐标
- 若点击位置不可用，例如未命中地图、命中 UI、超出允许区域，则不下发控制命令
- 新目标点下发后，应覆盖该无人机上一次未完成的临时目标，除非后续明确引入任务队列机制

#### 5.7.3 UI 反馈

- 当前选中的无人机应持续显示“已选中”状态
- 地图点击派发成功后，应在目标点位置显示落点标记
- UI 应显示当前控制对象，例如“当前控制：UAV-01”
- 派发失败时应有失败反馈，例如“未选中无人机”或“目标点无效”

#### 5.7.4 状态约束

- 离线、失联或不可控状态的无人机不应接受新的地图目标点派发
- 正在执行阵列序列的无人机是否允许被单独抢占控制，应在实现中作为明确规则处理
- 当系统处于自由镜头模式时，当前选中无人机状态仍应保留

### 5.8 验收标准

- 鼠标点击无人机后，可稳定选中目标且高亮显示
- 选中后再次点击地图，只有该无人机会收到目标点
- 点击其他无人机时，选中状态能正确切换
- 未选中无人机时点击地图，不会误发控制命令
- 点击无效区域时，系统不会产生错误动作

### FR-03 鼠标滚轮按下且悬停无人机时，弹出无人机信息 UI

### 5.9 目标

提供运行时无人机信息查看入口，并为未来接收更多来自无人机的数据字段预留扩展能力。

### 5.10 交互流程

1. 鼠标悬停在某一架无人机上
2. 操作员按下鼠标滚轮中键
3. 系统识别当前悬停目标为无人机
4. 弹出对应无人机的信息面板
5. 面板显示当前基础信息，并支持后续追加更多字段

### 5.11 详细需求

#### 5.11.1 触发条件

- 只有当鼠标当前悬停对象为无人机时，中键按下才触发信息面板
- 若鼠标未悬停在无人机上，中键按下不弹出无人机信息面板
- 若当前已有信息面板打开，再次对同一无人机触发时可刷新或聚焦已有面板

#### 5.11.2 信息面板内容

首版信息面板至少应支持以下内容：

| 字段 | 说明 |
|------|------|
| 无人机 ID / 名称 | 用于识别当前对象 |
| 在线状态 | 在线、离线、失联等 |
| 当前位置 | 世界坐标或地图坐标 |
| 当前高度 | 用于快速判断飞行层级 |
| 当前速度 | 可为标量或三轴速度 |
| 当前姿态 | 偏航、俯仰、横滚等 |
| 最近更新时间 | 便于判断数据是否实时 |

#### 5.11.3 扩展性要求

- 面板数据结构应支持后续新增字段，而不需要重做整体 UI 架构
- 后续可扩展字段包括但不限于电池电量、GPS 状态、链路质量、飞行模式、任务状态、故障告警等
- 当某字段当前暂无数据时，应显示默认占位文本，例如“暂无数据”

#### 5.11.4 面板行为

- 面板应明确展示其对应的是哪一架无人机
- 面板应支持关闭、切换目标或自动刷新
- 若无人机数据持续更新，面板中的信息应同步刷新
- 面板不应阻塞地图主交互，可根据实际设计采用悬浮窗、侧边栏或固定信息卡样式

### 5.12 验收标准

- 鼠标悬停无人机并按中键后，可稳定弹出该无人机的信息 UI
- 不悬停在无人机上时，中键不会误弹无人机面板
- 面板至少显示首版基础字段，并且字段缺失时有占位表现
- 同一套 UI 结构可继续扩展更多无人机数据字段

### FR-04 多机情况下，按 F 键切换为可自由移动的镜头模式

### 5.13 目标

当场景中存在多架无人机时，为操作员提供脱离无人机跟随视角的自由观察能力，便于基于当前视角方向移动镜头，统览地图、阵列和任务态势。

### 5.14 功能说明

在多机模式下，操作员可通过键盘操作将镜头从“跟随某架无人机”切换为“自由镜头”。自由镜头开启后，镜头可在地图空间中独立移动，不再跟随无人机位置变化；其中 `W/A/S/D` 的前后左右移动方向应相对于当前镜头视角定义。

### 5.15 详细需求

#### 5.15.1 模式切换

- 仅在多机场景下启用该能力
- 用户按下 `F` 键后，镜头从当前跟随状态切换为自由镜头状态
- 自由镜头状态下，当前已选中的无人机对象不应丢失
- UI 应显示当前镜头模式，例如“跟随镜头”或“自由镜头”

#### 5.15.2 自由镜头控制

- `W/A/S/D` 控制镜头相对于当前视角方向进行前后左右相对移动
- `Q/E` 控制镜头上下移动
- 鼠标移动控制镜头视角旋转
- 镜头移动应具备基本平滑性，避免明显抖动或突变

#### 5.15.3 跟随逻辑解除

- 切换到自由镜头后，镜头不再持续锁定某一架无人机
- 无人机继续飞行时，镜头位置不应被动跟随改变
- 自由镜头模式下仍可继续进行无人机选中、信息查看和地图点选派发

#### 5.15.4 返回跟随模式

- 再次点击`F`键从自由镜头返回跟随镜头。
- 返回后，镜头应优先跟随当前选中的无人机；若无选中对象，可跟随默认主控无人机
- 返回跟随时，应避免相机瞬移造成明显视觉突兀，可采用平滑过渡

### 5.16 验收标准

- 在多机模式下可成功切换到自由镜头
- 自由镜头状态下不再跟随无人机运动
- 用户可通过 `W/A/S/D` 相对于当前视角方向进行前后左右移动，配合 `Q/E` 与鼠标完成自由观察
- 自由镜头下仍可继续完成无人机选中和目标点派发
- 可从自由镜头稳定返回跟随模式

### FR-05 无人机路径编辑器：多机路径序列编辑与播放

### 5.17 目标

提供一个运行在 UE5 场景内的**路径编辑器**，让操作员可以在 Cesium 地图上为多架无人机分别绘制路径（一组有序航点），编辑完成后一键播放，系统按路径驱动真实无人机飞行。

首版以”可演示、可保存、可扩展”为原则，不要求实现复杂任务规划或碰撞避让。

---

### 5.18 核心概念定义

| 概念 | 定义 |
|------|------|
| 路径（Path） | 属于某一架无人机的有序航点列表，每个航点包含世界坐标和停留时间 |
| 航点（Waypoint） | 路径中的一个目标点，包含位置（UE5世界坐标）、停留时间（秒）、可选高度覆盖 |
| 路径序列（PathSequence） | 多架无人机各自路径的集合，是保存和播放的最小单元 |
| 预设（Preset） | 内置的路径序列模板，参数化生成，不可直接编辑但可克隆为自定义序列 |
| 播放（Playback） | 按时间顺序向各无人机下发航点目标，驱动真实飞机飞行 |

---

### 5.19 子任务拆解

FR-05 拆分为以下 5 个可独立分工的子任务。

---

#### T5-01 数据层：路径数据结构与存储

**负责人**：C++ 工程师（依赖 FR-01 T1-01 完成）

**目标**：定义路径编辑器的核心数据模型，提供保存和加载能力。

**需要新建**：

`FormationPresetDataAsset`（`UPrimaryDataAsset`）：
```cpp
struct FWaypoint {
    FVector WorldLocation;    // UE5 世界坐标（厘米），由 DroneCoordinateService 保证
    float DwellTimeSec;       // 到达后停留时间（秒），0 = 不停留直接飞下一个
    bool bOverrideAltitude;   // 是否覆盖高度
    float AltitudeOverride;   // 覆盖高度（米，相对地图中心海拔）
};

struct FDronePath {
    int32 DroneId;            // 对应无人机 ID
    TArray<FWaypoint> Waypoints;
    float CruiseSpeedMps;     // 巡航速度（米/秒），0 = 使用默认
};

struct FPathSequence {
    FString SequenceId;       // 唯一 ID（GUID）
    FString DisplayName;      // 显示名称
    TArray<FDronePath> DronePaths;
    bool bIsPreset;           // true = 内置预设，false = 用户自定义
    FString PresetType;       // 预设类型标识（”Line”/”Column”/”Square”/”Circle”/”Custom”）
};
```

`DroneOpsSaveGame`（`USaveGame`）：
- `TArray<FPathSequence> SavedSequences`
- `FString LastEditedSequenceId`
- 提供 `SaveSequence()`、`LoadAllSequences()`、`DeleteSequence()` 接口

**验收**：能保存一个包含 3 架无人机、每架 5 个航点的路径序列，重启后能正确加载

---

#### T5-02 预设生成器：内置路径预设

**负责人**：C++ 工程师（依赖 T5-01）

**目标**：提供参数化生成内置预设路径的工具函数，用户选择预设后自动生成各无人机的航点列表。

**需要新建**：`FormationPresetGenerator`（静态工具类）

提供以下预设生成函数，所有坐标通过 `DroneCoordinateService` 转换：

| 预设名称 | 参数 | 说明 |
|----------|------|------|
| 横向一字排开 | 基准点、间距（米）、高度（米） | N 架飞机沿 Y 轴（东西）等间距排列 |
| 纵向队列 | 基准点、间距（米）、高度（米） | N 架飞机沿 X 轴（南北）等间距排列 |
| 方阵 | 基准点、行列数、间距（米）、高度（米） | 最多支持 3×3 = 9 架 |
| 圆形展开 | 圆心、半径（米）、高度（米） | N 架飞机均匀分布在圆周上 |
| 整体平移 | 起点、终点、高度（米） | 所有飞机保持相对位置整体移动 |
| 整体升降 | 当前位置、目标高度（米） | 所有飞机垂直升降到指定高度 |

每个函数签名：
```cpp
static FPathSequence GenerateLine(
    TArray<int32> DroneIds,
    FVector BasepointWorld,   // 基准点 UE5 世界坐标
    float SpacingMeters,
    float AltitudeMeters,
    float DwellTimeSec = 3.0f
);
```

**验收**：给定 3 架无人机和基准点，`GenerateLine` 生成的 3 条路径各有 1 个航点，X 坐标相同，Y 坐标间距 = `SpacingMeters × 100`（厘米）

#### T5-03 编辑器 UI：`WBP_PathEditor`

**负责人**：UI 工程师（依赖 T5-01、T5-02，依赖 FR-02 的无人机选中能力）

**目标**：提供运行时路径编辑界面，支持在地图上点击添加航点、调整参数、管理序列。

**UI 布局**：

#### T5-04 播放引擎：`FormationPlaybackSubsystem`

**负责人**：C++ 工程师（依赖 T5-01，依赖 FR-02 的控制发送能力）

**目标**：实现路径序列的播放调度逻辑，按时间驱动各无人机飞向下一个航点。

**需要新建**：`FormationPlaybackSubsystem`（`UGameInstanceSubsystem`）

**核心状态机**：

```
Idle → Playing → (到达航点) → Dwelling → (停留结束) → Playing
                            → (最后航点) → Completed
Playing → Paused → Playing（恢复）
Playing/Paused → Stopped → Idle
```

**播放逻辑**：

1. `StartPlayback(FPathSequence, TArray<int32> ParticipantDroneIds)`
   - 验证所有参与无人机在线
   - 对每架无人机，记录当前航点索引 = 0
   - 向所有参与无人机发送第一个航点目标（通过 `DroneCommandSenderComponent`）
   - 对每架无人机启动独立的到达检测计时器

2. 到达检测：每帧检查无人机当前位置与目标航点的距离
   - 距离 < `ArrivalThresholdMeters`（默认 2m）视为到达
   - 到达后等待 `DwellTimeSec`，然后发送下一个航点

3. 所有无人机完成最后一个航点后，状态切换为 `Completed`

**对外接口**：
```cpp
void StartPlayback(const FPathSequence& Sequence, const TArray<int32>& DroneIds);
void PausePlayback();
void ResumePlayback();
void StopPlayback();
EFormationPlaybackState GetState() const;
float GetProgress() const;  // 0.0 ~ 1.0，基于已完成航点数
FOnPlaybackStateChanged OnStateChanged;  // 广播事件
```

**控制锁定规则**：
- 播放中的无人机进入 `EDroneControlLockReason::FormationPlayback` 锁定状态
- 锁定状态下，FR-02 的单机点选派发被拒绝，UI 显示”路径播放中”提示
- `StopPlayback()` 解除所有锁定，无人机切换为悬停

**验收**：
- 给定 2 架无人机各 3 个航点的序列，`StartPlayback` 后两架飞机按顺序飞向各自航点
- 飞行中调用 `PausePlayback()`，飞机停在当前位置（发送悬停命令）
- 调用 `ResumePlayback()`，飞机继续飞向当前航点

## 6. 非功能需求

### 6.1 可用性

- 主要交互应尽量减少操作步骤，避免频繁切换复杂模式
- 关键状态需有明确 UI 提示，包括选中状态、镜头模式、阵列播放状态、地图加载状态
- 错误信息应可读，避免只输出技术性日志而没有用户可理解反馈

### 6.2 性能

- 流式地图加载不应导致频繁长时间卡顿
- 多机状态显示、信息面板刷新和阵列播放不应明显拖慢主视角操作
- 在合理无人机数量范围内，镜头移动和鼠标交互应保持流畅

### 6.3 可扩展性

- 无人机信息面板需支持持续增加数据字段
- 阵列编辑功能需便于从“简单预设”扩展到“更复杂协同脚本”
- 地图能力需支持未来叠加更多业务图层

### 6.4 可维护性

- 地图配置、无人机配置、阵列预设应尽量数据化，而不是硬编码在单一逻辑中
- 核心交互状态应有清晰定义，例如选中态、自由镜头态、阵列执行态
